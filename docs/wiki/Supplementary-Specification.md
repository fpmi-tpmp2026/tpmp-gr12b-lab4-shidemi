# Дополнительная спецификация

## Надежность

- Включены внешние ключи SQLite.
- Ограничения целостности проверяют даты, пробег, количество пассажиров и стоимость билета.
- При добавлении рейса приложение проверяет существование автобуса и маршрута.
- Операции изменения и удаления рейсов доступны только администратору.

## Удобство использования

- Меню содержит короткие пункты по основным отчетам.
- В базе есть тестовые пользователи и демонстрационные данные.
- Ошибки ввода выводятся в консоль.

## Производительность

- Для учебного объема данных используется локальная SQLite БД.
- Отчеты выполняются SQL-запросами с агрегацией на стороне SQLite.

## Сопровождаемость

- Исходный код разделен на модули `db`, `auth`, `reports`, `main`.
- Сборка описана в `Makefile`.
- CI запускает сборку, тесты и покрытие.
- CRUD-операции рейсов выделены в модуль `reports.cpp`: `addTrip`, `updateTripPassengers`, `deleteTrip`.
